数据结构的基本概念

一些概念

数据

描述客观属性的数.字符

数据元素

数据元素是数据的基本单位,整体考虑(eg:一条学生记录)

一个数据元素由若干个数据项组成,数据项是数据元素不可分割的最小单位(eg:一条学生记录由学号,姓名,性别等构成)

数据类型

是指一组性质相同的值的集合及定义在此集合上的一些操作的总称。

原子类型: 不可分割(整形,字符型)

结构类型: 还可分解(数组)

抽象数据类型：是指一个数学模型及定义在该模型的上的一组操作。
1.数据对象集。
2.数据集合相关联的操作集。
抽象：描述数据结构类型的方法不依赖于具体实现。
1.与存放数据的机器无关。
2.与数据存储的物理结构无关。
3.与实现操作的算法和编译语言无关。

使用 (数据对象,数据关系,基本操作集)这样的三元组来表示

数据结构

所有的数据元素之间都存在着某种关系,我们将其称之为结构

可分为:

逻辑结构:与算法设计有关

1.集合:类似数学上的集合

2.线性结构:一对一

3.树形结构:一对多

4.图状结构或网状结构:多对多

存储结构:与算法的实现有关

顺序存储: 存储物理地址相邻

链式存储: 存储物理地址未必相邻,记录了相邻元素的物理位置

索引存储: 类似于目录

散列存储: 通过关键字计算出元素的物理地址

数据的运算

运算的定义(针对逻辑结构)

运算的实现(针对物理结构)

算法和算法的复杂度

算法定义

是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示一个或者多个操作。

算法的特性

1.输入：算法具有零个或者多个输入。
2.输出：至少有一个或者多个输出。
3.有穷性：指算法在执行 有限的步骤之后，自动结束，而不会出现无限循环，并且每一个步骤在可接受的时间内完成。
4.确定性：算法的每一个步骤都具有确定的含义，不会出现 二义性 。
5.可行性：算法的每一步都必须是可行的，也就是说，每一步都能够通过执行有限的次数完成。

算法的复杂度

空间复杂度S(n)

用来衡量随个算法规模的增大,算法所需空间增加的快慢(计算时可能开辟新的存储空间)

S(n)=O(g(n))

时间复杂度T(n)

用来衡量随个算法规模的增大,算法时间增加的快慢

主要分析的是T(n)的数量级

T(n)=O(f(n)),f(n)是算法中基本运算的频度 (语句执行频度表示一次运行中语句总共执行的次数) ,一般考虑最坏情况下时间复杂度.

计算方法:取f(n)增长最快的函数项,把他的系数改为1.

常见的时间复杂度所耗时间的大小排列：
$$O(1) < O(log(n)) < O(n) < O(nlog(n)) < O(n^2) < O(n^3) < O(2^n) < O(n!) <O(n^n)$$

时间复杂度的计算

1.忽略常数项

2.一般可以直接关注循环

单个循环体

多个循环体

加法规则: 选俩个循环里多的一个

乘法规则: 常用于嵌套循环

空间复杂度的计算

S(n)=O(g(n))

辅助空间: 除了存储算法本身的指令,常熟,变量和输入数据外,还需要存储对数据操作的存储单元

算法原地工作是指算法所需的辅助空间是常量,即O(1)
